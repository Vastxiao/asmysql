    简介¶

    asmysql 是一个基于 aiomysql 封装的异步 MySQL 客户端库，专为 Python 异步编程设计。v2 版本完全重构了架构，提供了更清晰、更灵活的 API 设计，支持类型提示、连接池管理、错误处理等企业级特性。

    主要优势¶

    • 简单易用：直观的 API 设计，学习成本低
    • 类型安全：完整的类型提示支持，提升开发体验
    • 高性能：基于连接池的异步操作，支持高并发
    • 灵活扩展：分离引擎和业务逻辑，便于架构设计
    • 内存友好：支持流式查询，处理大数据集不占用内存

    特性¶

    v2 版本核心特性¶

    1. 分离式架构
    2. Engine 类：独立的 MySQL 连接引擎，可单独使用
    3. AsMysql 类：业务逻辑开发基类，继承即可使用

    4. Result 类：结果处理类，支持多种数据获取方式

    5. 连接管理

    6. 自动管理 MySQL 连接池
    7. 支持连接池配置（最小/最大连接数）
    8. 自动重连机制

    9. 连接状态监控

    10. 灵活的查询方式

    11. 支持普通查询和流式查询
    12. 支持单条和批量执行

    13. 支持事务控制

    14. 多种结果类型

    15. tuple：默认元组类型
    16. dict：字典类型

    17. 自定义模型：支持 Pydantic 等模型类

    18. 数据获取方式

    19. fetch_one()：获取单条记录
    20. fetch_many()：获取多条记录
    21. fetch_all()：获取所有记录
    22. iterate()：异步迭代器，逐行获取

    23. 直接迭代：async for item in result

    24. 错误处理

    25. 全局自动捕获 MysqlError
    26. 错误码和错误信息访问

    27. 优雅的错误处理机制

    28. 上下文管理

    29. 支持 async with 语法
    30. 自动资源清理

    31. 支持异步迭代器协议

    32. URL 连接字符串

    33. 支持 MySQL URL 格式连接
    34. 格式：mysql://user:password@host:port/?charset=utf8mb4

    核心概念¶

    Engine（引擎）¶

    Engine 是 MySQL 连接的核心类，负责：

    • 管理连接池
    • 执行 SQL 语句
    • 处理连接生命周期

    AsMysql（业务逻辑类）¶

    AsMysql 是业务逻辑开发的基类，提供：

    • client 属性：访问 Engine 实例
    • 业务方法封装
    • 代码组织能力

    Result（结果类）¶

    Result 是 SQL 执行结果的封装类，提供：

    • 数据获取方法
    • 错误信息访问
    • 执行统计信息
    • 异步迭代支持
